## Requirements

## 2.1 Funktionale Requirements

| Nr. | Requirement |
|-----|----------------------------------|
| F1 | Einstellung einer **Soll-Drehzahl in 10 %-Schritten (0–100 %)**, Änderungen innerhalb von **1 s** im Istwert übernommen |
| F2 | **Not-Halt-Befehl** erzeugt **STOP**-Zustand, Stillstand (Istwert = 0 %) innerhalb von **100 ms** |
| F3 | Aktualisierung der **Monitoringdaten** im **500 ms** Intervall, Berechnet eines geglätteten Stromwert (Mittelwert über 5 Messungen). |
| F4 | **Wartungshinweis** generiert und im Log erfasst, Intervall **48 Stunden kumulierter Laufzeit** |
| F5 | Zyklische Protokollierung im **1 Hz**-Takt in **CSV-Datei**, Erfassung von Zeit, Sollwert, Istwert, Betriebszustand und Stromschätzung |

---

## 2.2 Nicht-funktionale Requirements

| **Kriterium** | **Zielwert** | **Beschreibung** |
|----------------|--------------|------------------|
| Reaktionszeit Benutzerbefehl → Zustandsänderung | ≤ 200 ms | Umsetzung eines Bedienbefehls (Start, Stop, Not-Halt) in der Steuerlogik |
| Aktualisierungsintervall Messwerte | 500 ms | Zyklische Erfassung und Berechnung der Stromaufnahme |
| Genauigkeit Drehzahlanzeige | ± 5 % | Abweichung zwischen Soll- und Ist-Drehzahl |
| Wartungsintervall | 48 h kumulierte Laufzeit | Generierung eines Wartungshinweises nach Erreichen der Betriebszeit |
| Dateigröße Logdatei | ≤ 1 MB | Automatisches Erstellen einer neuen CSV-Datei bei Überschreitung der Größe |
| Programmiersprache | C++ | Objektorientierte Implementierung auf Mikrocontroller-Basis (z. B. STM32) |
| Umgebungstemperatur | 0–45 °C | Sicherstellung der Funktion unter typischen Haushaltsbedingungen |
